# 浏览器多进程架构

## 一句话定义

浏览器多进程架构是一种把页面渲染、GPU、网络等能力拆到独立进程中的运行方式，用进程隔离换取稳定性、安全性和可恢复性。

## 它解决什么问题

早期单进程浏览器里，任意页面崩溃、死循环或恶意脚本都可能拖垮整个浏览器。多进程架构把不同职责拆开后，单个页面或单类服务出问题时，影响范围可以被限制在局部。

## 核心机制

### 1. 进程级隔离

Chromium 会把页面渲染、GPU、网络与基础服务拆到不同进程中运行。这样一个标签页崩溃，通常不会直接带崩其他标签页和浏览器主进程。

### 2. 渲染进程内部的线程分工

一个渲染进程里，常见的关键线程有四类：

- **主线程**：解析 HTML、CSS，执行 JavaScript，做布局、分层和绘制指令生成
- **合成线程**：接收图层树，切分图块，调度光栅化，组装合成帧并提交给 GPU
- **光栅化线程池**：把绘制指令转换成位图，通常并行工作
- **Web Worker**：在后台执行耗时 JS 计算，不参与渲染

### 3. 合成线程与光栅化线程的分工

| 维度 | 合成线程 | 光栅化线程 |
|---|---|---|
| 数量 | 每个渲染进程通常 1 个 | 线程池，通常多个 |
| 角色 | 调度与组装 | 像素填充 |
| 输入 | 图层树、绘制指令 | 图块任务 |
| 输出 | 合成帧 | 位图 |
| 擅长场景 | 滚动、`transform` 动画调度 | 图块并行绘制 |

合成线程负责把图层切成图块，并按优先级分发给光栅化线程池。光栅化线程完成像素绘制后，结果再回到合成线程，由它统一组装后交给 GPU。

### 4. 图层与图块是两层概念

- **图层（Layer）**：主线程在布局和分层阶段形成的逻辑渲染单元
- **图块（Tile）**：合成线程把图层切开的物理处理单元，常见尺寸是 256×256 或 512×512 像素

图层决定“谁单独参与合成”，图块决定“如何并行光栅化”。

### 5. 位图与光栅化

位图是按像素点阵存储图像的表示方式，每个像素保存 RGBA 颜色值。光栅化就是把矢量绘制指令逐像素计算成位图的过程，结果才能被 GPU 直接显示。

## 完整渲染流程

1. **主线程**：解析 HTML 和 CSS，构建 DOM / CSSOM，完成布局、分层、绘制指令生成
2. **合成线程**：接收图层树，把图层切成图块并按优先级调度
3. **光栅化线程池**：把图块的绘制指令转成位图
4. **合成线程**：收集位图，组装为合成帧
5. **GPU 进程**：把合成帧显示到屏幕

## 内存控制思路

Chromium 为了控制多进程带来的额外内存，会采用几类常见策略：

- **共享渲染进程**：部分同源页面可能复用同一个 renderer process
- **后台换出**：标签页进入后台后，系统会积极回收可释放内存
- **基础服务复用**：网络等公共服务放在共享进程中，避免每个页面各自持有一份

## 对前端开发的启示

- **不要假设后台页面一直活跃**：页面切到后台后，定时器节奏、资源占用和生命周期都可能变化，适合用 `visibilitychange` 做状态恢复
- **主线程阻塞会直接卡渲染**：长任务会阻塞布局、绘制、事件处理和 JS 执行
- **滚动与 `transform` 动画优先走合成路径**：这类更新更容易避开主线程上的重布局与重绘
- **大响应适合流式处理**：网络请求可能不在渲染进程里跑，但大对象解析和持有仍会吃掉页面内存
- **组件卸载时要清理引用**：事件监听器、闭包、大对象引用和 Detached DOM 都可能造成泄漏
- **跨窗口隔离要留意 `rel="noopener"`**：新页面与原页面的引用关系会影响隔离边界

## 延伸知识点

### Web Worker

Web Worker 适合承接耗时计算，通过 `postMessage` 与主线程通信，但不能直接访问 DOM、CSSOM 和 `window`。

### V8 垃圾回收

V8 主要基于可达性判断对象是否存活：从根对象出发仍能访问到的对象会被保留，访问不到的对象会被回收。

- **新生代**：偏向高频回收短生命周期对象
- **老生代**：偏向低频回收长生命周期对象

前端里常见的内存泄漏，本质上往往是对象仍然能从根引用链被访问到。

## 常见误区

- **“渲染流程就是单向流水线”**：更贴近事实的理解是以合成线程为中心的调度结构，任务会在合成线程和光栅化线程之间往返
- **“合成线程负责算 CSS 层级”**：层叠关系、`z-index` 和 stacking context 的计算主要发生在主线程
- **“Web Worker 能帮忙渲染页面”**：它只能分担计算，不能参与 DOM 渲染
- **“页面不卡就说明主线程没压力”**：合成线程可以让部分动画和滚动继续顺滑，但主线程仍可能已经拥堵

## 适用场景

- 理解浏览器渲染性能问题
- 分析滚动卡顿、动画掉帧与长任务阻塞
- 理解为什么某些 CSS 属性更适合做动画
- 解释前端内存泄漏和后台标签页行为
