## 当前理解边界
- 我已经理解：React 项目结构需要先局部内聚，再在稳定复用后提升公共抽象；组件拆分应以语义和交互周期为边界；BFF / controller 的核心是让 View 面对稳定、拿来即用的数据形状；请求封装要把防抖、缓存、取消重复请求、失败重试放在统一入口中治理。
- 我还不理解：大型 React 项目中 Redux、zustand、React Query、BFF 分层之间的具体取舍边界；不同团队规模下 controller 粒度如何保持一致；请求治理与 React Query / SWR 这类数据请求库之间的职责边界。
